SqlMap日常使用相关

sqlmap最基本命令

远程连接数据库并打印 -f DBMS信息 –banner版本信息

sqlmap直接对单一URL探测，参数使用-u或–url，现在查询的是数据库版本信息

-m 从多行文本文件中读取多个目标，对多个目标进行探测类似-u

sqlmap通过-g自动利用Google获取指定Google hack的目标，利用交互向导模式进行sql注入探测

sqlmap会自动在探测过程中使用适当的HTTP请求方法，但有些方法不会自动使用如PUT，所以需要强制指定使用 –method=PUT

sqlmap 默认使用的HTTP请求为GET，如上指定为POST或者隐式地将其改为POST如：

覆盖参数–data里的默认连接符，如指定&为分隔符(此处没啥用，因为默认就是&)

sqlmap设置请求参数

sqlmap设置Cookie参数

  • 使用场景：

    • Web应用程序具有基于Cookie验证的过程
    • 想利用Cookie值上的SQL注入漏洞

  • sqlmap使用Cookie过程：

    • 截包，复制Cookie
    • sqlmap使用 –cookie粘贴Cookie

sqlmap指定–user-Agent或者使用自带的随机头–random-agent,因为一些网站会识别sqlmap默认HTTP请求头,为避免出发安全机制

在针对user-agent的值进行SQL注入探测时，需要指定–level 3或者用*指定探测位置，否则不会去探测user-agent

sqlmap设置HTTP主机头值，想要探测主机头是否存在SQL注入需–level 5或加*

sqlmap设置HTTP Referer值，想要探测Referer值是否存在SQL注入需–level 3或加*

sqlmap设置额外的HTTP标头 –headers=”user-agent:HAHA\nCookie:1234555”注入\n

salmap中设置代理的参数：–proxy，–proxy-cred,可以隐藏自己真实IP

设置tor参数隐藏自己：–tor,–tor-port,–tor-type,–check-tor

sqlmap设置延迟–delay，为了避免封IP

sqlmap设置超时，默认30秒，–timeout 10.5

sqlmap设置重试次数，默认3次，–retries 4

sqlmap设置随机化参数，默认情况会随机化，–randomize id可以避免触发安全机制

sqlmap设置不进行URL编码，因为一些服务器不遵循RFC标准，需要以原始非编码形式发送值–skip-urlencode

sqlmap设置持久连接–keep-alive,能节约一点服务器资源

sqlmap设置多线程，默认1为了不影响服务器性能，最高10，–thread 10效率

sqlmap设置空连接，表示检索HTTP中的没有Body的响应，常用在盲注中，–null-connection效率

控制对哪一个参数进行sql注入探测：-p id 或 -p “id,uname”也可以*指定（用在URL内部注入）

sqlmap设置注入参数

指定数据库管理系统DBMS，–dbms mysql 5.1.1

指定后端操作系统os,–os windows \ –os linux

sqlmap强制设置无效值(原id=12)替换,–invalid-bignum替换超大值，–invalid-logical同and 1=2,–invalid-string同id=asdsdf

sqlmap设置tamper脚本绕WAF

sqlmap设置风险参数–risk 1-3 ，风险2,3可能导致被测试单位数据或其它东西出问题

sqlmap设置具体sql注入技术：

  • B：布尔盲注
  • E：报错盲注
  • U：联合查询注入
  • S：堆叠注入
  • T：时间盲注
  • Q：内联查询注入

sqlmap设置union查询表，数据库权限不足以使攻击者查询其它表或库，可以指定具体数据表获取一定的信息 –union-from 表名

  • 获取DBMS当前用户

    • –current-user

  • 检索DBMS的当前数据库名

    • –current-db

  • 获取当前DBMS用户名

    • –hostname

  • sqlmap探测当前用户DBA

    • –is-dba

  • sqlmap枚举DBMS所有用户

    • –users

  • 获取用户密码

    • –password

  • 获取当前用户对DBMS的权限

    • –privileges
    • –role

  • –dbs列举数据库名

  • 枚举数据库表

    • –tables
    • 指定数据库表-D 数据库名
    • exclude-sysdbs非系统数据库的表名

  • 枚举数据表的列

    • –columns
      • -D指定数据库
      • -T指定数据表
      • -C指定具体字段

  • 枚举数据信息

    • –dump

  • –schema用于检索DBMS模式，模式列表包含所有数据库、表、列以及他们各自类型

  • –count检索数据表数量可指定数据库

  • sqlmap暴力破解表名列名（在mysql<5.0时无元数据库information_scehma)
    • –common-tables -D 数据库
    • –common-columns

  • sqlmap读取文件

    • –file-read=”绝对路径”

  • sqlmap写入文件

    • –file-write 读取本地文件
    • –file-dest 将读取的文件写入到远程得绝对路径里

  • sqlmap检索所有信息

    • -a 或者–all，检索以上命令答案

sqlmap系统参数

  • –os-shell

    • 利用sql漏洞sqlmap自动写入文件，利用文件反弹shell以执行cmd命令

  • sqlmap修改注册表

    • –reg-read
    • –reg-add
    • –reg-del
    • –reg-key…
